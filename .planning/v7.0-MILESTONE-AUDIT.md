## Integration Gaps (Non-Blocking)

### INT-01: Chat-to-Visualizer Navigation (Deferred by Design)

The chat page (`page.tsx`) calls `/api/generate` exclusively and never routes to `/visualizer`. This was acknowledged in comments as future scope. The visualizer is a complete, self-contained system — it works when the store is hydrated (via tests or direct API call).

**Fix scope (v7.5):** Add "View in Visualizer" button to chat page after generation; call `/api/preview` and `store.hydrate()` before navigating.

### INT-02: Controller/Footswitch Data Flow

`extractControllerAssignments()` and `extractFootswitchAssignments()` exist and are tested but never called in the `/api/preview` route. The route calls `hydrateVisualizerState(presetSpec, deviceTarget)` with only 2 args — no controller data is extracted from the built file.

**Fix scope (v7.5):** Build the HLX file in preview route, extract controller sections, pass to `hydrateVisualizerState()`.

### INT-03: Footswitch Badge Prop Not Passed

`SignalChainCanvas` never reads `footswitchAssignments` from store or passes `footswitchAssignment` prop to `BlockTile`. The FS badge rendering code in `BlockTile` is correct but unreachable.

**Fix scope (v7.5):** Add `getFootswitchForBlock()` call in `SignalChainCanvas` rendering path, pass result as prop to `BlockTile`.
