## Milestone Summary

**Key Decisions:**
- logUsage() is a no-op when LOG_USAGE !== true — zero production performance impact
- usage.jsonl JSON-lines format for append-only, parse-friendly token log
- Variax is NOT a signal chain block — it's an input configuration (@input: 3 = Multi)
- Pod Go and Stadium don't support VDI — isVariaxSupported() guard excludes them
- Chat AI NEVER asks about Variax unprompted (reactive only)
- Stadium dual-amp uses single-amp fallback (dual-amp unsupported)
- Phase 52 (Stadium XL) deferred to v4.0

**Issues Resolved:**
- Stadium Agoura amp lookup failure in chain-rules
- buildBlockKeyMap dual-DSP key collision (dsp1 block0 overwriting dsp0 block0)
- Fire-and-forget DB update errors silently discarded in generate route
- VARIAX_MODEL_NAMES missing from barrel exports

**Issues Deferred to v4.0:**
- Phases 43-47: Core quality improvements (planner prompt, amp params, FX, combos, cost routing)
- Stadium preset builder rebuild from real .hsp files
- Stadium device selection temporarily blocked in UI

**Technical Debt:**
- No SUMMARY.md for phases 49 and 51 (implemented without formal planning)
