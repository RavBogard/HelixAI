# Milestone v2.0: Persistent Chat Platform

**Status:** ✅ SHIPPED 2026-03-04
**Phases:** 24-30
**Total Plans:** 16

## Overview

Transform HelixAI from a stateless generate-and-download tool into a persistent platform where users log in with Google, maintain a sidebar of past conversations, pick up where they left off, and re-download their most recent preset per chat. Anonymous usage remains fully functional; login unlocks history.

### Phase 26: Conversation CRUD API

**Goal**: A stable, independently-testable data API exists for conversations and messages — all CRUD operations work correctly, RLS verified by cross-user isolation, sequence numbers enforce correct message ordering
**Depends on**: Phase 24
**Requirements**: CONV-01, CONV-02, CONV-03, CONV-04, CONV-05, CONV-06
**Plans**: 2/2 complete — completed 2026-03-03

Plans:

- [x] 26-02: `POST /api/conversations/[id]/messages` (save with server-side sequence_number); `DELETE /api/conversations/[id]` (with storage cleanup) (CONV-05, CONV-06)

**Details:**

- Title truncated server-side to 60 chars — client cannot bypass

- Server-side sequence number via `MAX+1` query before insert
- Read-then-delete ordering for storage cleanup (preset_url read before row deletion)

## Milestone Summary

**Milestone Goal Achievement:** ✅ Goal achieved (94% at audit; 100% after UXP-01 fix)

**Key Decisions:**
- Anonymous-first auth model — non-logged-in users get full functionality; login unlocks persistence only
- `linkIdentity()` for anonymous-to-authenticated upgrade — preserves UUID across OAuth redirect
- Save last preset per chat (not all versions) — balances utility with storage cost
- Supabase for auth/database/storage — single isomorphic SDK, generous free tier
- CSS `translateX` sidebar toggle — no remount, no re-fetch on open/close
- Custom window events for cross-component comms — AuthButton lives in Server Component layout; state lives in Client Component page.tsx
- `conversationIdRef` (useRef) for synchronous access to async-updated conversationId — avoids React state race condition on first message
- Defense-in-depth: every API route independently verifies session (against CVE-2025-29927)
- 404 not 403 for non-owned resources — no existence leakage
- Dual-amp schema uses Zod `.refine()` — schema-level enforcement of `secondCabName` requirement

**Issues Resolved:**
- UXP-01 / MISS-01: Sign-in banner `onClick` dispatched state serialization event but never triggered OAuth redirect — fixed by replicating `AuthButton.handleSignIn()` logic inline
- INFRA-06 scope discrepancy: Google OAuth registration deferred to Phase 25 (AUTH-01) — functionally resolved

**Issues Deferred (Tech Debt):**
- MINOR-01: `ConversationList.deviceLabel()` falls through to `.toUpperCase()` for `helix_stomp`/`helix_stomp_xl`/`helix_stadium` — cosmetic, fixed in later phases

- MINOR-03: `/api/generate` does not verify conversation ownership before storage upload — mitigated by RLS + user_id prefix path
