  <instructions>
The user must complete these steps (Claude cannot access external dashboards):

1. **Create Supabase project** (if not already done):
   - Go to https://supabase.com/dashboard
   - Click "New Project", choose a name and region
   - Wait for project to provision (~2 minutes)

2. **Run the schema SQL:**
   - Go to Supabase Dashboard -> SQL Editor -> New query
   - Paste the entire contents of `supabase/schema.sql`
   - Click "Run"
   - Verify the output shows no errors

3. **Copy environment variables:**
   - Go to Supabase Dashboard -> Project Settings -> API
   - Copy "Project URL" into `.env.local` as `NEXT_PUBLIC_SUPABASE_URL`
   - Copy "anon/public" API key into `.env.local` as `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - Generate a CRON_SECRET: `openssl rand -hex 16` (or any random 32-char string)
   - Add `CRON_SECRET=<generated-value>` to `.env.local`

4. **Google OAuth callback URL** (preparation for Phase 25):
   - Go to Supabase Dashboard -> Authentication -> Providers -> Google
   - Copy the "Callback URL" shown (it looks like `https://<project-ref>.supabase.co/auth/v1/callback`)
   - NOTE: You do NOT need to configure the Google Cloud Console OAuth app yet. That will be done in Phase 25 when Google Auth is implemented. For now, just note this URL exists for future reference.

5. **Set Vercel environment variables** (for deployment):
   - Go to Vercel Dashboard -> Project -> Settings -> Environment Variables
   - Add `NEXT_PUBLIC_SUPABASE_URL` for Preview + Production
   - Add `NEXT_PUBLIC_SUPABASE_ANON_KEY` for Preview + Production
   - Add `CRON_SECRET` for Preview + Production

6. **Verify locally:**
   - Run `npm run dev`
   - Visit `http://localhost:3000/api/test-session` in browser
   - Expected: `{"user":null,"error":"..."}` (user is null because no one is logged in — this is correct)
   - The key is that the route does NOT crash with a configuration error
