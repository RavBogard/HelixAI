  <files>src/lib/visualizer/state-diff.ts, src/lib/visualizer/state-diff.test.ts</files>
  <behavior>
    - No changes: calculateStateDiff(original, current) returns { hasChanges: false, chainChanges: [], modelSwaps: [], snapshotChanges: [] } when original === current (deep equal)
    - Block reordered: When a block moves from position 2 to position 0 on same DSP, chainChanges includes { blockId, from: { dsp, position, path }, to: { dsp, position, path } }
    - Block moved cross-DSP: When a block moves from DSP 0 to DSP 1, chainChanges includes the DSP change
    - Model swap: When block at amp0 changes modelId from "HD2_AmpBR1" to "HD2_AmpBR2" (position stays same), modelSwaps includes { blockId, fromModelId, toModelId }
    - Block added: When current has a block not in original, chainChanges includes { type: "added", block: BlockSpec }
    - Block removed: When original has a block not in current, chainChanges includes { type: "removed", blockId }
    - Snapshot parameter override: When snapshots[1].parameterOverrides.delay1.Mix changes, snapshotChanges[1].parameterOverrides includes the change
    - Snapshot bypass change: When snapshots[0].blockStates.reverb0 flips from true to false, snapshotChanges[0].blockStates includes the change
    - Mixed changes: Multiple types of changes in a single diff all captured correctly
    - hasChanges is true when any chainChanges, modelSwaps, or snapshotChanges are non-empty
  </behavior>
  <action>
    Create `src/lib/visualizer/state-diff.ts` exporting:

    1. `StateDiff` interface:
    ```typescript
    export interface StateDiff {
      hasChanges: boolean;
      chainChanges: ChainChange[];
      modelSwaps: ModelSwap[];
      snapshotChanges: SnapshotChange[];
    }
    export type ChainChange =
      | { type: "moved"; blockId: string; from: { dsp: number; position: number; path: number }; to: { dsp: number; position: number; path: number } }
      | { type: "added"; block: BlockSpec }
      | { type: "removed"; blockId: string };
    export interface ModelSwap { blockId: string; fromModelId: string; toModelId: string }
    export interface SnapshotChange {
      index: number;
      blockStates: Record<string, boolean>;
      parameterOverrides: Record<string, Record<string, number | boolean>>;
    }
    ```

    2. `calculateStateDiff(original, current)` function:
    - Compare baseBlocks by generating blockId from type+position: iterate original blocks and current blocks.
    - For each block in original: if not in current (by matching modelId+type or position), record as "removed".
    - For each block in current: if not in original, record as "added".
    - For matched blocks (same blockId in both): check if dsp/position/path changed (chainChange "moved") or if modelId changed (modelSwap).
    - For snapshots: iterate indices 0-3, compare blockStates and parameterOverrides key-by-key. Only include changed keys in the SnapshotChange.
    - Set hasChanges = true if any array is non-empty.

    Note: Block identity matching strategy — match by stable blockId (type+position) from the ORIGINAL state. When blocks are reordered, their positions change, so compare by index in the original array vs the current array. Use a two-pass approach: first pass builds a map of original blocks by index, second pass matches current blocks. Unmatched originals = removed, unmatched current = added.

    IMPORTANT: The diff only needs to detect what changed, not be a patch format. The download endpoint receives the FULL current state — the diff is used by the frontend to decide whether to send the full state or signal "no changes" (download the original).
  </action>

    Export `dehydrateToPresetSpec()`:
    ```typescript

      baseBlocks: BlockSpec[],
      snapshots: SnapshotSpec[],
      meta: { presetName: string; description: string; tempo: number }
    ): PresetSpec
    ```
    - signalChain = baseBlocks (identity — the blocks ARE the signal chain, same as hydrate direction)
    - name = meta.presetName, description = meta.description, tempo = meta.tempo
    - snapshots = snapshots array (pass through — builders handle the snapshot format)

    **Part B: Create `src/app/api/download/route.ts`**

    POST handler accepting JSON body:
    ```typescript
    { device: DeviceTarget, baseBlocks: BlockSpec[], snapshots: SnapshotSpec[],
      presetName: string, description: string, tempo: number }
    ```

    Steps:
    1. Validate required fields (device, baseBlocks, snapshots, presetName).
    2. Call `dehydrateToPresetSpec()` to reconstruct PresetSpec.
    3. Call `validatePresetSpec(presetSpec, caps)` — fail with 400 if invalid.
    4. Branch on device type using existing isStomp/isStadium/isPodGo guards:
       - Stomp: buildStompFile() -> JSON.stringify -> Buffer
       - Stadium: buildHspFile() -> hspFile.serialized (already binary string)
       - Pod Go: buildPgpFile() -> JSON.stringify -> Buffer
       - Helix (default): buildHlxFile() -> JSON.stringify -> Buffer
    5. Return Response with:
       - Content-Type: application/octet-stream
       - Content-Disposition: attachment; filename="{presetName}.{ext}"
       - File extension: .hlx (Helix/Stomp), .pgp (Pod Go), .hsp (Stadium)

    Error handling: try/catch wrapping the pipeline, return 500 with error message on failure. Return 400 for missing required fields.

    Note: No persistence in download route — the /api/generate route handles Supabase storage. The download endpoint is stateless: frontend sends state, backend compiles file, returns binary.
