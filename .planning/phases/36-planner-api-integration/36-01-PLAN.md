1. Add `buildHspFile` and `summarizeStadiumPreset` to imports from `@/lib/helix`
2. Replace the Stadium 501 block with the real Stadium path:
   - `buildHspFile(presetSpec)` → get `hspFile`
   - `summarizeStadiumPreset(presetSpec)` → `summary`
   - Fire-and-forget upload to `latest.hsp`
   - Return `{ preset: hspFile.json, summary, fileExtension: ".hsp", ... }`
3. The `.hsp` serialized content is `hspFile.serialized` — but the API returns the JSON object (not the serialized string with magic header). The frontend and Supabase storage will receive `hspFile.json`. The magic header is added when writing to disk — the generate route stores `hspFile.serialized` as a Buffer for Supabase (so the downloaded .hsp file has the header).

## Key Decision: What to Store

- **Supabase Storage**: store `hspFile.serialized` (magic header + JSON) as Buffer — this is the file users download
- **API response `preset` field**: send `hspFile.json` (pure JSON object) — consistent with .hlx and .pgp which send JSON objects
