**Step 2: Create device-family.ts**

Create a new file `src/lib/helix/device-family.ts` containing:

1. `DeviceFamily` type — string literal union: `"helix" | "stomp" | "podgo" | "stadium"`

2. `DeviceCapabilities` interface with these fields:
   - `family: DeviceFamily`
   - `dspCount: number` — Number of DSP chips (helix: 2, others: 1)
   - `maxBlocksPerDsp: number` — Max effect blocks per DSP chip
   - `maxBlocksTotal: number` — Total blocks across all DSPs
   - `maxSnapshots: number` — Snapshot count limit per preset
   - `dualAmpSupported: boolean` — Can the device run two amp blocks?
   - `pathCount: number` — Number of stereo signal paths
   - `variaxSupported: boolean` — VDI input present
   - `sendReturnCount: number` — Hardware FX loop count
   - `expressionPedalCount: number` — Built-in + external expression pedal inputs
   - `fileFormat: "hlx" | "pgp" | "hsp"` — Preset file extension
   - `ampCatalogEra: "hd2" | "agoura"` — Which amp model era this family uses
   - `availableBlockTypes: ReadonlyArray<BlockSpec["type"]>` — Block types valid for this device

3. `assertNever(x: never): never` — throws Error with descriptive message

4. `resolveFamily(device: DeviceTarget): DeviceFamily` — exhaustive switch over all 9 DeviceTarget values with `default: assertNever(device)`. Mapping:
   - helix_lt, helix_floor, helix_rack → "helix"
   - helix_stomp, helix_stomp_xl → "stomp"
   - pod_go, pod_go_xl → "podgo"
   - helix_stadium, helix_stadium_xl → "stadium"

5. Capability constants (NOT exported — private to module):
   - `ALL_BLOCK_TYPES`: all 12 BlockSpec types
   - `POD_GO_BLOCK_TYPES`: excludes "pitch" and "send_return" (Pod Go FX loop is fixed, not user-assignable)
   - `HELIX_CAPABILITIES`: family "helix", dspCount 2, maxBlocksPerDsp 8, maxBlocksTotal 32, maxSnapshots 8, dualAmpSupported true, pathCount 4, variaxSupported true, sendReturnCount 4, expressionPedalCount 3, fileFormat "hlx", ampCatalogEra "hd2", all block types
   - `STOMP_CAPABILITIES`: family "stomp", dspCount 1, maxBlocksPerDsp STOMP_CONFIG.STOMP_MAX_BLOCKS (6), maxBlocksTotal STOMP_CONFIG.STOMP_MAX_BLOCKS (6), maxSnapshots STOMP_CONFIG.STOMP_MAX_SNAPSHOTS (3), dualAmpSupported false, pathCount 1, variaxSupported true, sendReturnCount 1, expressionPedalCount 2, fileFormat "hlx", ampCatalogEra "hd2", all block types
   - `STOMP_XL_CAPABILITIES`: same as STOMP but maxBlocksPerDsp/maxBlocksTotal from STOMP_CONFIG.STOMP_XL_MAX_BLOCKS (9), maxSnapshots from STOMP_CONFIG.STOMP_XL_MAX_SNAPSHOTS (4)
   - `POD_GO_CAPABILITIES`: family "podgo", dspCount 1, maxBlocksPerDsp POD_GO_MAX_USER_EFFECTS (4), maxBlocksTotal 10, maxSnapshots 4, dualAmpSupported false, pathCount 1, variaxSupported false, sendReturnCount 1, expressionPedalCount 1, fileFormat "pgp", ampCatalogEra "hd2", POD_GO_BLOCK_TYPES
   - `STADIUM_CAPABILITIES`: family "stadium", dspCount 1, maxBlocksPerDsp 48, maxBlocksTotal 48, maxSnapshots STADIUM_CONFIG.STADIUM_MAX_SNAPSHOTS (8), dualAmpSupported true, pathCount 4, variaxSupported false, sendReturnCount 2, expressionPedalCount 0, fileFormat "hsp", ampCatalogEra "agoura", all block types
   - Add comment on STADIUM_CAPABILITIES: "Stadium and Stadium XL share capabilities. XL has 4 FX loops and 1 built-in EXP vs Stadium's 2 FX loops and 0 EXP. Using conservative shared values. Split into STADIUM_XL_CAPABILITIES if per-device precision needed."

6. `getCapabilities(device: DeviceTarget): DeviceCapabilities` — exhaustive switch over all 9 DeviceTarget values with `default: assertNever(device)`. Maps:
   - helix_lt, helix_floor, helix_rack → HELIX_CAPABILITIES

   - helix_stadium, helix_stadium_xl → STADIUM_CAPABILITIES (share caps)

Import `type { DeviceTarget, BlockSpec }` from `./types` and `{ STOMP_CONFIG, STADIUM_CONFIG }` from `./config` and `{ POD_GO_MAX_USER_EFFECTS }` from `./types`.
