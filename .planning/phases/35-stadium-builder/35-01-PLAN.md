## Objective

- `{ "meta": {..., "device_id": 2490368}, "preset": {"flow": [...], "snapshots": [...], ...} }` structure

- Export from index.ts barrel
- DO NOT modify preset-builder.ts or podgo-builder.ts

## Key Format Details (from stadium-device-id.md, Phase 31)

- Magic header: `rpshnosj` (8 ASCII bytes prepended to JSON)
- Top level: `{ meta: {...}, preset: {...} }` — NOT `{ data: {...} }`
- `meta.device_id`: 2490368
- `meta.device_version`: 301990022
- `preset.flow`: array of path objects (each path = object of block keys)
- `preset.snapshots`: array of 8 snapshot objects
- Block keys: `b00`, `b01`, `b02`, ... (not `block0`, `block1`, ...)
- I/O models: `P35_InputInst1`, `P35_AppDSPSplitY`, `P35_AppDSPJoin`, `P35_OutputMatrix`

## Implementation Plan

### Task 1: Create stadium-builder.ts

Public API:
- `buildHspFile(spec: PresetSpec): { magic: string; json: object; serialized: string }`
  - Returns object with magic header + JSON content + full serialized string
  - serialized = magic header + JSON.stringify(json)
- `summarizeStadiumPreset(spec: PresetSpec): string`

Internal structure:
- `buildStadiumMeta(spec)` — builds the meta object
- `buildStadiumPreset(spec)` — builds the preset object with flow + snapshots + params
- `buildStadiumFlow(spec)` — builds preset.flow array (single path for v3.0)
- `buildStadiumSnapshots(spec)` — builds preset.snapshots array (8 slots)

### Task 2: Export from index.ts

Add `export { buildHspFile, summarizeStadiumPreset } from "./stadium-builder";`
