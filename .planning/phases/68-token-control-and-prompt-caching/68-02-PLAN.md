<objective>
Unify the Stomp family planner prompt so helix_stomp and helix_stomp_xl produce byte-identical system prompt text, sharing a single Anthropic cache entry. Move device-specific restrictions (block count, snapshot count, device name) to the user message in planner.ts. This matches the existing Helix pattern where helix_lt and helix_floor share one cache entry.

Purpose: Currently helix_stomp and helix_stomp_xl produce different system prompt text (different device name, block count, snapshot count interpolated into the prompt body), creating two separate cache entries. This doubles cold-write cost ($6.00/MTok each) and halves cache hit rate for the Stomp family. Unification consolidates to one cache entry.

Output: Unified stomp/prompt.ts, planner.ts with Stomp variant restriction in user message, updated tests.
</objective>
