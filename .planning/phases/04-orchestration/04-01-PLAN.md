  <name>Task 2: Create strict validatePresetSpec and wire into generate route</name>
  <files>src/lib/helix/validate.ts, src/lib/helix/index.ts, src/app/api/generate/route.ts</files>
  <action>
1. In `src/lib/helix/validate.ts`, add a new `validatePresetSpec(spec: PresetSpec): void` function that throws on structural errors instead of auto-correcting. Keep the existing `validateAndFixPresetSpec` unchanged (it may be useful for debugging). The new function validates:

   a. **Signal chain not empty**: `if (spec.signalChain.length === 0) throw new Error("PresetSpec has empty signal chain")`
   b. **At least one amp block**: `if (!spec.signalChain.some(b => b.type === "amp")) throw new Error("PresetSpec missing amp block")`
   c. **At least one cab block**: `if (!spec.signalChain.some(b => b.type === "cab")) throw new Error("PresetSpec missing cab block")`
   d. **All model IDs valid**: For each block in signalChain, check `VALID_IDS.has(block.modelId)`. If any invalid: `throw new Error("Invalid model ID '${block.modelId}' for block '${block.modelName}'")`
   e. **Snapshots present**: `if (!spec.snapshots || spec.snapshots.length === 0) throw new Error("PresetSpec has no snapshots")`
   f. **Parameter ranges**: For each block's parameters, validate type-appropriate ranges:
      - Cab Mic: integer 0-15
      - Cab LowCut: 19.9-500.0 Hz
      - Cab HighCut: 1000.0-20100.0 Hz
      - All other params: 0.0-1.0 normalized
      If out of range: `throw new Error("Parameter '${key}' value ${value} out of range for block '${block.modelName}'")`
   g. **DSP block limits**: Count non-cab blocks per DSP. If either DSP has more than 8: `throw new Error("DSP${dsp} exceeds 8-block limit (${count} blocks)")`

   This function is intentionally strict. The Knowledge Layer should produce valid specs, so any validation failure indicates a bug, not normal operation.

2. In `src/lib/helix/index.ts`, add export for `validatePresetSpec` alongside the existing `validateAndFixPresetSpec` export.

   - This will throw if anything is structurally wrong, and the existing try/catch will convert it to a 500 error response.
   - Extract `device` from the request body: `const { messages, device } = await req.json();`

   - Pass `deviceTarget` to `buildHlxFile`: `const hlxFile = buildHlxFile(presetSpec, deviceTarget);`
   - Include `device: deviceTarget` in the JSON response so the frontend knows which device was targeted.
