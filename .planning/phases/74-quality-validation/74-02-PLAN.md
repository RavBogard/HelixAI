       - Define 6 devices: `["helix_lt", "helix_floor", "helix_stomp", "helix_stomp_xl", "pod_go", "helix_stadium"]`
       - Define 6 genres: `["blues", "rock", "metal", "jazz", "ambient", "country"]`
       - Create fixed `GENRE_INTENTS` — one deterministic ToneIntent per genre. Each must specify:
         - `ampName`: A known amp for the genre (e.g., blues="US Deluxe Nrm", rock="Brit 2204", metal="Revv Gen Red", jazz="US Double Nrm", ambient="Matchstick Ch1", country="US Deluxe Nrm")
         - `cabName`: Matching cab
         - `genreHint`: The genre string
         - `guitarType`: "single_coil" for blues/country/jazz, "humbucker" for rock/metal/ambient
         - `effects`: 2-3 effects per genre (use known model names from the codebase)
         - `snapshots`: 4 standard snapshots `[{role:"clean"},{role:"crunch"},{role:"lead"},{role:"ambient"}]`
         - `presetName`: `"baseline-{genre}"`
         - `description`: Brief description
       - For Stadium device, create alternate intents using Agoura amp names (check STADIUM_AMPS in models.ts for valid names). Use a helper `buildGenreIntent(genre, caps)` that returns the correct ToneIntent based on `caps.ampCatalogEra`.
       - For each device x genre combination:
         1. `const caps = getCapabilities(device)`
         2. Build ToneIntent via `buildGenreIntent(genre, caps)`
         3. `const chain = assembleSignalChain(intent, caps)`
         4. `const parameterized = resolveParameters(chain, intent, caps)`
         5. `const snapshots = buildSnapshots(parameterized, intent.snapshots)`
         6. Construct PresetSpec
         7. `const warnings = validatePresetQuality(spec, caps)`
         8. Record: `{ device, genre, warningCount, warningCodes, warnings }`
       - Wrap each device x genre in try/catch — if any combination fails, log the error and continue (some device/amp combos may not work perfectly).
