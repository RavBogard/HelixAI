CRITICAL REACTIVITY BUG:
Both SignalChainCanvas.tsx and ParameterEditorPane.tsx currently call
`useVisualizerStore.getState()` to get the full store state for computed selectors.
This is an IMPERATIVE read — it does NOT cause React re-renders when state changes.
When activeSnapshotIndex changes, these components do NOT re-render, so bypass states
and parameter values appear stale.

FIX: Subscribe to activeSnapshotIndex and snapshots reactively so React re-renders
when snapshots switch. For computed selectors that need the full state, use
`useVisualizerStore()` (the hook) instead of `.getState()`, or subscribe to the
specific dependencies that should trigger re-render.

2. Fix SignalChainCanvas.tsx reactivity:

   **Current problem:** Line ~358: `const state = useVisualizerStore.getState();` is imperative.
   The DspRow and layout components receive this `state` object and call `getEffectiveBlockState(state, blockId)`.
   When `activeSnapshotIndex` changes, `state` is stale because getState() was called once at render time and the component doesn't re-render.

   **Fix approach:** Subscribe to `activeSnapshotIndex` and `snapshots` reactively so the component re-renders when they change. The simplest fix:

   ```tsx
   // ADD these reactive subscriptions (they trigger re-render on change):
   const activeSnapshotIndex = useVisualizerStore((s) => s.activeSnapshotIndex);
   const snapshots = useVisualizerStore((s) => s.snapshots);

   // Keep getState() for the full object — but now the component re-renders
   // when activeSnapshotIndex or snapshots change, so getState() returns fresh data
   const state = useVisualizerStore.getState();
   ```

   This works because React re-renders the component when any subscribed selector value changes. After re-render, getState() returns the latest state.

   Also subscribe to `baseBlocks` reactively (it may already be subscribed — verify):
   ```tsx
   const baseBlocks = useVisualizerStore((s) => s.baseBlocks);
   ```

   The key insight: we need the reactive subscription for the values that determine visual output (activeSnapshotIndex, snapshots, baseBlocks). The `state` from getState() is fine to pass to computed selectors as long as it's called during a render triggered by the reactive subscriptions.
