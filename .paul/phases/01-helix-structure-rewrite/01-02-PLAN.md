<objective>
## Goal
Fix the three quality issues surfaced during Plan 01-01 HX Edit verification: (1) delay tempo sync using TempoSync1/SyncSelect1 instead of broken Time formula, (2) per-snapshot amp Drive control so CLEAN actually sounds clean, (3) ambient snapshot sound design with higher mix values and longer decays.

## Purpose
Users expect preset snapshots to sound meaningfully different — clean should be glassy, ambient should be atmospheric, and delays should lock to the requested subdivision. Current output fails all three: the amp breaks up on clean, ambient is just "delays turned on," and dotted 8th notes calculate to ~152ms (a straight 8th) instead of ~304ms.

## Output
- param-engine.ts: delays emit TempoSync1/SyncSelect1 params, Time formula fixed
- snapshot-engine.ts: amp Drive controlled per snapshot role, ambient mix/decay values increased
- preset-builder.ts: amp Drive added to controller section for snapshot control
- All 1201+ tests still passing
</objective>

## Key Findings from Reference Analysis
1. Real delays have: TempoSync1 (boolean), SyncSelect1 (integer subdivision), Time (seconds fallback)
   - SyncSelect1 values: 6=8th note, 8=dotted eighth, 9=16th note
   - When TempoSync1=true, hardware handles sync — Time is fallback only
2. Real presets control amp Drive via snapshot controllers (min/max range)
   - Strab reference: dsp0/block4/Drive min=0.350, max=0.550, @controller=13
3. Time formula bug: `30/BPM` = 8th note, NOT quarter note. Should be `60/BPM` for quarter note in seconds.
   At 148 BPM: 30/148*0.75 = 0.152s (wrong), 60/148*0.75 = 0.304s (correct dotted 8th)

  <files>src/lib/helix/param-engine.ts</files>
  <action>
    1. FIX the Time formula (FX-03 section, ~line 601):
       - Change `30 / tempoHint` to `60 / tempoHint`
       - This gives quarter note time in SECONDS (not the current half-quarter)
       - Verify: at 148 BPM, dotted 8th = 60/148 * 0.75 = 0.3041s ✓

    2. ADD TempoSync1 and SyncSelect1 to delay block parameter resolution:
       - In resolveDefaultParams() or a new helper, when a delay block is resolved:
         - Add `TempoSync1: true` when tempoHint is present (enable hardware sync)
         - Add `SyncSelect1: subdivisionToSyncSelect(delaySubdivision)` mapping:
           - "quarter" → 4
           - "dotted_eighth" → 8
           - "eighth" → 6
           - "triplet" → 5
           - Default → 6 (eighth note)
         - Keep Time as fallback value (correctly calculated)
       - When tempoHint is NOT present:
         - Add `TempoSync1: false`
         - Add `SyncSelect1: 6` (default eighth, like reference presets)
         - Time uses genre defaults as before

    3. VERIFY SyncSelect1 mapping against reference presets:
       - Alchemy Sultan 2: SyncSelect1=6 (8th note) with TempoSync1=false
       - Strab ORNG: SyncSelect1=8 (dotted 8th) with TempoSync1=true on some delays

    Avoid: Changing genre default Time values (those are fine for non-synced mode).
    Avoid: Touching Left Time/Right Time dual delay logic (different pattern).
  </action>
  <verify>

       - @max: ROLE_DRIVE.lead (0.60) — highest value across snapshots
       - This tells HX Edit that Drive is snapshot-controlled
