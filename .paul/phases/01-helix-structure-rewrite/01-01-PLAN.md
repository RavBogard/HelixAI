<objective>
## Goal
Rewrite the Helix preset builder's DSP structure to match the exact patterns from real HX Edit exports. Every generated .hlx file must include inputB, outputB, split, join on BOTH DSPs, correct routing models, and all required metadata sections (dt, powercab, variax).

## Purpose
Users report presets fail to load or behave incorrectly in HX Edit because generated .hlx files are structurally incomplete compared to real factory/user presets. This plan adopts a "golden preset" methodology — reverse-engineer from real exports, match exactly.

## Output
- preset-builder.ts rewritten to emit structurally complete DSP objects
- types.ts updated to make required fields non-optional
- snapshot-engine.ts updated to include split in snapshot block entries
- All 1201+ existing tests still passing
</objective>

2. inputA always uses HD2_AppDSPFlow1Input, inputB always uses HD2_AppDSPFlow2Input — regardless of which DSP
3. When DSP0 routes to DSP1: dsp0.outputA.@output=2, dsp1.inputA.@input=0
4. outputB.@output is always 0
5. split model: HD2_AppDSPFlowSplitY, join model: HD2_AppDSPFlowJoin
6. dt0/dt1/dtdual, powercab0/1/dual, variax sections always present
7. Snapshot blocks include split state

