# Phase 20 Plan 01: DSP Block Budget Enforcement

**Bug:** Helix Floor presets with high-gain amps can exceed the 8-block DSP1 limit because COMBO-03 truncation is skipped (`maxEffectsPerDsp: Infinity`) and mandatory blocks (EQ, Gain Block, Horizon Gate) are inserted *after* truncation without rechecking the budget.

**Trigger scenario:** Rudolf Schenker 1980s tone → 2203 amp (high_gain) → planner requests 6+ post-cab effects → 3 mandatory DSP1 blocks added → 9 total → crash.

**Fix:** Replace the throwing validation at step 8 in `assembleSignalChain` with a graceful per-DSP budget enforcement that drops lowest-priority user effects until within limits.

## Scope

**Single file change:** `src/lib/helix/chain-rules.ts`
**Test file:** `src/lib/helix/chain-rules.test.ts`

## Tasks

### Task 1: Replace DSP block limit validation with budget enforcement

**File:** `src/lib/helix/chain-rules.ts`

**Current behavior (step 8, lines 657-688):**
After assembling all blocks (user effects + mandatory + amp/cab), the code counts non-cab blocks per DSP and throws if either exceeds `caps.maxBlocksPerDsp`.

**New behavior:**
After assembling all blocks at step 7 (sort), replace the throwing validation with a loop that:

1. Count non-cab blocks on each DSP
2. While any DSP exceeds `caps.maxBlocksPerDsp`:
   - Identify user-effect blocks on the over-budget DSP (exclude mandatory blocks and amp)
   - Score them using the existing `getEffectPriority()` function
   - Drop the lowest-priority user effect from `allBlocks`
   - Log a warning: `[chain-rules] DSP{N} budget exceeded: dropping "{effectName}" ({intentRole}) to fit {max}-block limit`
3. Re-sort remaining blocks

**How to distinguish user effects from mandatory blocks:**
- Mandatory blocks have slots: `"boost"`, `"horizon_gate"`, `"eq"`, `"gain_block"`
- User effects have slots: `"wah"`, `"compressor"`, `"extra_drive"`, `"modulation"`, `"delay"`, `"reverb"`
- Amp and cab have slots: `"amp"`, `"cab"`

Create a `const MANDATORY_SLOTS` set for clarity.

**Boundaries:**
- Do NOT change `maxEffectsPerDsp` from `Infinity` — the budget enforcement handles it
- Do NOT change the mandatory block insertion logic (CHAIN-06)
- Do NOT change `getDspForSlot` or `SLOT_ORDER`
- Keep the single-DSP validation path (line 658-666) unchanged — only modify the dual-DSP path

### Task 2: Add tests

**File:** `src/lib/helix/chain-rules.test.ts`

Add tests:

1. **High-gain Helix preset with 6 post-cab effects does not throw** — verify it produces a valid chain with <= 8 blocks on DSP1, and the lowest-priority effect is dropped
2. **Clean/crunch Helix preset with 7 post-cab effects** — only 2 mandatory DSP1 blocks (no Horizon Gate), so drops start at 9 total. Verify graceful truncation
3. **Preset within budget is unchanged** — verify a normal preset (3 user effects + mandatory) passes through without any drops
