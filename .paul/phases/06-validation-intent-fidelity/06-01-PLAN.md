<objective>
## Goal
Add an intent fidelity validation layer that verifies ToneIntent choices made it into the final PresetSpec, plus an intent audit object returned in the API response so users can see what was applied.

## Purpose
The Knowledge Layer pipeline (chain-rules → param-engine → snapshot-engine) transforms ToneIntent into PresetSpec through multiple deterministic steps. Currently, nothing validates that the user's requested amp, cab, effects, tempo, delay subdivision, and snapshot count actually survived the pipeline. This phase adds that traceability — zero AI cost, purely deterministic.

## Output
- `intent-validate.ts`: Intent fidelity validation functions (advisory, non-throwing)
- `intent-validate.test.ts`: Tests for all validation rules
- Updated generate route: returns `intentAudit` object in API response
</objective>

## AC-1: Amp/Cab Intent Fidelity
```gherkin
Given a ToneIntent with ampName "US Double Nrm" and cabName "4x12 Greenback25"
When the PresetSpec is validated against the ToneIntent
Then the audit confirms amp and cab models match
And if either is missing from the signal chain, a warning is raised
```

## AC-2: Effect Intent Fidelity
```gherkin
Given a ToneIntent with effects [{ modelName: "Glitz", role: "ambient" }, { modelName: "Scream 808", role: "always_on" }]
When the PresetSpec is validated against the ToneIntent
Then the audit lists each requested effect with matched/missing status
And missing effects generate a warning with the effect name
```

## AC-3: Tempo and Delay Subdivision Fidelity
```gherkin
Given a ToneIntent with tempoHint 140 and delaySubdivision "dotted_eighth"
When the PresetSpec is validated against the ToneIntent
Then the audit confirms tempo matches (±1 BPM tolerance)
And the audit confirms delay blocks have TempoSync1 enabled
And if tempo or delay subdivision mismatches, a warning is raised
```

## AC-4: Snapshot Count Fidelity
```gherkin
Given a ToneIntent with 4 snapshots
When the PresetSpec is validated against the ToneIntent
Then the audit confirms PresetSpec has at least 4 snapshots
And if fewer snapshots than requested, a warning is raised
```

```gherkin
  - delaySubdivision: { requested, applied }
  - snapshots: { requested, actual, matched }

```

  <name>Task 1: Create intent fidelity validation module</name>
  <files>src/lib/helix/intent-validate.ts, src/lib/helix/intent-validate.test.ts, src/lib/helix/index.ts</files>
  <action>
    **Create `src/lib/helix/intent-validate.ts`:**

    1. Define IntentAudit interface:
       ```
       interface IntentAuditEntry { requested: string; matched: boolean; actual?: string }
       interface EffectAuditEntry { requested: string; role: string; matched: boolean }
       interface IntentAudit {
         amp: IntentAuditEntry;
         cab: IntentAuditEntry;
         effects: EffectAuditEntry[];
         tempo: { requested: number; actual: number; matched: boolean };
         delaySubdivision: { requested: string | null; applied: boolean };
         snapshots: { requested: number; actual: number; matched: boolean };
         warnings: string[];
       }
       ```

    2. Create `auditIntentFidelity(toneIntent, presetSpec): IntentAudit`:
       - **Amp check:** Compare toneIntent.ampName against spec.signalChain amp block modelName
       - **Cab check:** Compare toneIntent.cabName against spec.signalChain cab block modelName
       - **Effects check:** For each toneIntent.effects[], check if a block with matching modelName exists in spec.signalChain. Use case-insensitive comparison. Record matched/missing status and role.
       - **Tempo check:** Compare toneIntent.tempoHint (or default 120) against spec.tempo. Match if within ±1 BPM.
       - **Delay subdivision check:** If toneIntent.delaySubdivision is set, check if any delay block has TempoSync1 parameter set to true. Record applied/not-applied.
       - **Snapshot count check:** Compare toneIntent.snapshots.length against spec.snapshots.length. Match if actual >= requested.
       - Build warnings[] from any mismatches.

    3. Export IntentAudit type and auditIntentFidelity function.

    4. Add export to `src/lib/helix/index.ts` barrel file.

    **Create `src/lib/helix/intent-validate.test.ts`:**

    5. Test cases using vitest (follow quality-validate.test.ts patterns):
       - Amp matches → no warning
       - Amp mismatch → warning with model names
       - Cab matches → no warning
       - Cab mismatch → warning
       - All effects present → no warnings
       - Missing effect → warning with effect name
       - Tempo matches → no warning
       - Tempo mismatch (>1 BPM) → warning
       - Delay subdivision applied → no warning
       - Delay subdivision requested but no TempoSync1 → warning
       - Snapshot count sufficient → no warning
       - Snapshot count insufficient → warning
       - Full audit returns correct IntentAudit shape

    Avoid: Adding any throws — this is advisory only (like quality-validate.ts)
    Avoid: Modifying validate.ts or quality-validate.ts
    Avoid: Any UI changes
  </action>

  </verify>
