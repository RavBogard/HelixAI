  <name>Task 1: Fix Pod Go block key mismatch</name>
  <files>src/lib/helix/podgo-builder.ts, src/lib/helix/podgo-builder.test.ts</files>
  <action>
    Fix `buildPgpBlockKeyMap()` at line 300-315:

    **Current bug:** The cab skip (`if (block.type === "cab") continue;`) happens AFTER `dspIdx++`, meaning cab blocks increment the DSP index but are excluded from snapshot keys. This creates a mismatch: snapshot block2 maps to DSP block3 instead of block2.

    **Fix:** Restructure the loop so cab blocks are skipped BEFORE incrementing dspIdx. Both indices should skip cabs identically:

    ```typescript
    for (const block of allBlocks) {
      if (block.type === "cab") continue; // skip cabs for BOTH indices
      const snapshotKey = `block${snapshotIdx}`;
      const dspKey = `block${dspIdx}`;
      map.set(snapshotKey, dspKey);
      snapshotIdx++;
      dspIdx++;
    }
    ```

    Wait — if both indices skip cabs identically, the map becomes identity (block0→block0). That means the map itself may be unnecessary, OR the original intent was to handle a different indexing scheme. Investigate:
    1. Read `buildPgpControllerSection()` (line 345-429) to understand how the mapped keys are used
    2. Read how Pod Go .pgp files actually index blocks (do they include cabs in block numbering?)
    3. If Pod Go .pgp format includes cabs in block indices but snapshot engine doesn't, then `dspIdx` should count ALL blocks (including cabs) but `snapshotIdx` should skip cabs — which is the CURRENT behavior
    4. In that case, the bug is the OPPOSITE: `dspIdx++` should come BEFORE the cab skip, and the current code may actually be correct

    **Resolution approach:** Write a test that creates a Pod Go preset with a cab in the chain, then verify:
    - snapshot engine's parameterOverrides use the correct block keys
    - podgo-builder's controller section references the correct blocks
    - The key mapping produces correct controller entries

    If the current logic IS correct (dspIdx counts cabs, snapshotIdx doesn't), then the audit finding was wrong — document as verified-correct.
    If the current logic IS buggy, apply the fix and add regression test.

    Avoid: Changing snapshot-engine.ts block key generation. Avoid changing the Pod Go .pgp format structure.
  </action>
