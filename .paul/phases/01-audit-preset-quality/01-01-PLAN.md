<objective>
## Goal
Comprehensive audit of preset generation quality across all 4 device families (Helix, Stomp, Pod Go, Stadium). Identify every issue causing presets to not "just work" on real hardware — broken parameters, incorrect model IDs, bad signal chain ordering, snapshot problems, gain staging errors, and format compliance failures.

## Purpose
This is the foundation for v1.0 Production Release. We can't fix what we haven't cataloged. This audit produces a prioritized issue list that drives Phases 2-4.

## Output
- `.paul/phases/01-audit-preset-quality/01-01-AUDIT-REPORT.md` — complete quality report with categorized issues, severity, and affected device families
</objective>


```gherkin
Given the 4 device-specific builders (HLX, PGP, HSP, Stomp)

```

<task type="auto">
  <name>Task 1: Run test suite and catalog failures</name>
  <files>all test files under src/</files>
  <action>
    Run full vitest suite. For each failure:
    - Record test name, file, and error message
    - Identify root cause (which module is broken)
    - Classify severity: critical (preset won't load), high (preset sounds wrong), medium (suboptimal), low (cosmetic)

    Also check test coverage gaps:
    - Which modules lack tests entirely?
    - Which device families have the least coverage?
    - Are there edge cases not tested (dual-amp, single-coil + high-gain, etc.)?
  </action>
  <verify>Test results captured with pass/fail counts per module</verify>
  <done>AC-1 satisfied: Complete test baseline with all failures cataloged</done>
</task>

<task type="auto">
  <name>Task 2: Audit model databases, chain rules, and parameter engine</name>
  <files>src/lib/helix/models.ts, src/lib/helix/chain-rules.ts, src/lib/helix/param-engine.ts, src/lib/helix/snapshot-engine.ts</files>
  <action>
    **Models audit:**
    - Cross-reference amp model IDs against Line 6 model list (check for typos, missing models, wrong categories)
    - Verify defaultParams are reasonable for each amp type (non-MV amps should have Master=1.0, etc.)
    - Check cab model IDs and mic assignments
    - Verify effect model IDs and parameter ranges
    - Check Stadium Agoura models separately (different era)

    **Chain rules audit:**
    - Trace assembleSignalChain() for each amp category (clean, crunch, high_gain) on each device family
    - Verify mandatory block insertion (EQ, Gain Block, Horizon Gate) per device
    - Check dual-DSP split/join routing for Helix
    - Verify single-DSP routing for Stomp/Pod Go/Stadium
    - Check for DSP block count limit compliance

    **Parameter engine audit:**
    - Verify 3-layer override system (model → category → per-model) resolves correctly
    - Check non-MV amp detection (Master forced to 1.0)
    - Verify guitar-type EQ adjustments
    - Check topology-aware parameter adjustments (cathode_follower vs plate_fed)
    - Verify cab parameter values (LowCut, HighCut, Mic) per category

    **Snapshot engine audit:**
    - Verify volume balancing across tone roles (clean/crunch/lead/ambient)
    - Check block state toggling logic (boost ON in lead, compressor OFF in lead, etc.)
    - Verify LED color assignments
    - Check ambient snapshot mix boost (+15%) for delay/reverb
  </action>
  <verify>All modules reviewed with issues documented per subsystem</verify>
  <done>AC-2 and AC-3 satisfied: Model accuracy and signal chain/parameter correctness audited</done>
</task>
