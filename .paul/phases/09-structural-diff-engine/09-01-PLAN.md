<objective>
## Goal
Create a deterministic structural diff engine that compares two preset JSON objects (generated vs reference) and produces a structured deviation report identifying every difference — at the JSON structure level, parameter values, snapshot behavior, controller mappings, metadata fields, and block configuration.

## Purpose
The v5.0 automated gold standard compliance pipeline needs a deep comparison framework. Phase 8 generates presets; this phase builds the comparison infrastructure. Phase 11 will feed real reference presets through this engine; Phase 12 runs the full audit. The diff engine must work across all 4 preset formats (HLX, PGP, HSP) with device-aware comparison logic.

## Output
- `src/lib/helix/structural-diff.ts` — Diff engine: deep JSON comparison with categorized deviation reports
- `src/lib/helix/structural-diff.test.ts` — Tests proving the diff engine correctly detects structural, parameter, metadata, snapshot, and controller differences across all preset formats
</objective>

## AC-1: Deep JSON Structural Diff
```gherkin
Given two preset JSON objects (same device family)
When the diff engine compares them
Then it produces a list of deviations, each containing:
  - path (JSON path to the differing field, e.g., "data.tone.dsp0.block2.@model")
  - category (structure | parameter | metadata | snapshot | controller | footswitch | block)
  - expected value (from reference)
  - actual value (from generated)
  - severity (critical | warning | info)
And identical presets produce zero deviations
```

## AC-2: Category-Aware Comparison
```gherkin
Given two presets with differences across multiple categories
When the diff engine compares them
Then deviations are correctly categorized:
  - "structure": missing/extra keys, wrong nesting, missing blocks
  - "parameter": different numeric parameter values on blocks
  - "metadata": device ID, device_version, name, appversion differences
  - "snapshot": snapshot name, validity, block states, parameter overrides
  - "controller": different @controller assignments, missing controllers
  - "footswitch": different FS assignments, missing footswitch entries
  - "block": different @model, @type, @position, @enabled, @stereo
```

## AC-3: Device-Family Aware Diffing
```gherkin
Given the diff engine receives a DeviceFamily hint
When comparing HLX presets (Helix/Stomp)
Then it knows to check dsp0/dsp1, inputA/outputA, cab0/cab1, split/join, 8 snapshots
When comparing PGP presets (Pod Go)
Then it knows to check input/output (not inputA), block0-block9, 4 snapshots, no @path/@stereo
When comparing HSP presets (Stadium)
Then it knows to check json.preset.flow[0].b00-b13, slot-based params, sources, snapshots array
```

## AC-4: Severity Classification
```gherkin
Given a set of deviations
When severity is assigned
Then:
  - critical: wrong device ID, missing amp/cab block, wrong block model, missing snapshots
  - warning: different parameter values, missing effects, different controller assignments
  - info: metadata differences (name, build_sha, appversion), cosmetic field ordering
```

## AC-5: Summary Report Generation
```gherkin
Given a completed diff comparison
When a summary report is generated
Then it includes:
  - total deviation count by severity (critical/warning/info)
  - total deviation count by category
  - pass/fail verdict (pass = zero critical deviations)
  - the full list of deviations sorted by severity then path
```

<task type="auto">
  <name>Task 1: Create structural diff engine</name>
  <files>src/lib/helix/structural-diff.ts</files>
  <action>
    Create the diff engine module with the following exports:

    **Types:**
    ```
    type DeviationCategory = "structure" | "parameter" | "metadata" | "snapshot" | "controller" | "footswitch" | "block";
    type DeviationSeverity = "critical" | "warning" | "info";

    interface Deviation {
      path: string;           // JSON path, e.g., "data.tone.dsp0.block2.@model"
      category: DeviationCategory;
      severity: DeviationSeverity;
      expected: unknown;      // Value from reference
      actual: unknown;        // Value from generated
      message: string;        // Human-readable description
    }

    interface DiffReport {
      deviceFamily: DeviceFamily;
      deviationCount: { critical: number; warning: number; info: number };
      categoryCount: Record<DeviationCategory, number>;
      passed: boolean;        // true if zero critical deviations
      deviations: Deviation[];
    }
    ```

    **Core function:**
    ```
    function diffPresets(
      reference: unknown,     // The reference preset JSON object
      generated: unknown,     // The generated preset JSON object
      family: DeviceFamily    // Device family for format-aware comparison
    ): DiffReport;
    ```

    **Implementation approach:**

    1. **Generic deep diff** (`deepDiff`): Recursively walk both objects. For each key:
       - Key exists in reference but not generated → "structure" deviation
       - Key exists in generated but not reference → "structure" deviation (extra key)
       - Both exist, values differ → classify based on path
       - Both exist, values equal → no deviation
       - For arrays: compare by index (not set-matching)

    2. **Path-based classification** (`classifyDeviation`): Given a JSON path, determine category and severity:
       - Paths containing "device" or "device_version" or "meta." → metadata
       - Paths containing "@model" or "@type" or "@position" or "@enabled" or "@stereo" or "@path" → block
       - Paths containing "snapshot" → snapshot
       - Paths containing "controller" → controller
       - Paths containing "footswitch" or "sources" → footswitch
       - Paths matching known parameter patterns (numeric leaf values on blocks) → parameter
       - Everything else → structure

    3. **Severity assignment** (`classifySeverity`):
       - critical: device ID mismatch, missing/extra amp or cab block (@model contains "Amp" or "Cab"), snapshot count mismatch, missing entire dsp/flow section
       - warning: parameter value differences, different @controller values, different footswitch assignments, missing non-amp/cab blocks
       - info: name, build_sha, appversion, modifieddate, cosmetic metadata

    4. **Family-specific entry points**: The main `diffPresets` function dispatches based on family:
       - `helix` / `stomp` / `stompXL` → `diffHlxPresets(ref, gen)` — walks data.tone.dsp0/dsp1, snapshots 0-7, controller, footswitch
       - `podGo` → `diffPgpPresets(ref, gen)` — walks data.tone.dsp0 with block0-block9, snapshots 0-3
       - `stadium` → `diffHspPresets(ref, gen)` — walks json.preset.flow[0].b00-b13, snapshots array, sources

       Each family-specific differ calls the generic `deepDiff` on the relevant sub-trees, then classifies all results.

    5. **Report builder** (`buildReport`): Aggregate deviations into DiffReport with counts and pass/fail verdict.

    **Key design decisions:**
    - Ignore key ordering (JSON objects are unordered) — compare by key name not position
    - For numeric comparisons: use tolerance of 0.001 for floating-point parameter values
    - Truncate very long values in `message` (e.g., serialized strings) to 100 chars
    - Do NOT attempt to "normalize" or reformat — compare raw JSON as-is

    **Avoid:**
    - Do NOT import from planner, chat, or any AI module
    - Do NOT make this async — pure synchronous comparison
    - Do NOT add file I/O — presets are in-memory objects
    - Do NOT try to parse .hlx/.pgp/.hsp file strings — this compares parsed JSON objects
  </action>

  <name>Task 2: Create comprehensive diff engine tests</name>
  <files>src/lib/helix/structural-diff.test.ts</files>
  <action>
    Create tests using vitest that prove the diff engine works correctly.

    **Test strategy:** Use the mock harness to generate real preset objects, then mutate them to create known differences for comparison.

    **Test groups:**

    1. **Identical preset comparison** (AC-1 baseline):
       - Generate a preset via `runScenario(helixCleanScenario)`
       - Deep-clone it
       - `diffPresets(original, clone, "helix")` → zero deviations, passed=true

    2. **Metadata differences** (AC-2: metadata, AC-4: info severity):
       - Clone a helix preset, change `data.meta.name` to "Modified"
       - Diff → 1 deviation, category="metadata", severity="info"
       - Clone, change `data.device` to 9999999
       - Diff → 1 deviation, category="metadata", severity="critical"

    3. **Block differences** (AC-2: block, AC-4: critical/warning):
       - Clone a preset, change `data.tone.dsp0.block0.@model` to "HD2_FakeAmp"
       - Diff → 1 deviation, category="block", severity="critical" (amp model change)
       - Clone, change `data.tone.dsp0.block2.@enabled` from true to false
       - Diff → 1 deviation, category="block", severity="warning"

    4. **Parameter differences** (AC-2: parameter, AC-4: warning):
       - Clone a preset, change a numeric parameter value on a block (e.g., Drive from 0.5 to 0.8)
       - Diff → 1 deviation, category="parameter", severity="warning"
       - Change a value by less than 0.001 (within tolerance) → no deviation

    5. **Snapshot differences** (AC-2: snapshot):
       - Clone a preset, change `snapshot0.@name` to "Modified"
       - Diff → 1 deviation, category="snapshot"
       - Delete `snapshot2` entirely → critical deviation (missing snapshot)

    6. **Controller differences** (AC-2: controller):
       - Clone a preset, modify a controller value
       - Diff → 1 deviation, category="controller"

    7. **Structure differences** (AC-2: structure):
       - Clone a preset, add an extra key that doesn't exist in reference
       - Diff → 1 deviation, category="structure"
       - Delete a key that exists in reference → 1 deviation

    8. **Pod Go format** (AC-3):
       - Generate a Pod Go preset via `runScenario(podGoCleanScenario)`
       - Clone and mutate
       - Verify diffs detect changes in Pod Go's block0-block9 format, 4-snapshot structure

    9. **Stadium format** (AC-3):
       - Generate a Stadium preset via `runScenario(stadiumCleanScenario)`
       - Clone and mutate
       - Verify diffs detect changes in Stadium's flow[0].b00-b13 format, slot-based params

    10. **Report summary** (AC-5):
        - Create a preset with multiple differences across categories
        - Verify DiffReport has correct counts by severity and category
        - Verify `passed` is false when critical deviations exist
        - Verify `passed` is true when only warning/info deviations exist

    **Import scenarios from mock-scenarios.ts** — use MOCK_SCENARIOS to find the right device/style combos. Use `runScenario` from mock-harness.ts to generate presets.

    **Helper:** Create a `deepClone(obj)` utility (JSON parse/stringify roundtrip) for creating mutated copies.

    **Avoid:**
    - Do NOT test every possible parameter — test representative cases
    - Do NOT create mock preset objects by hand — generate them via the harness
    - Do NOT test file I/O or string parsing
