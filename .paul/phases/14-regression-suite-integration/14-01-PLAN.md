## Goal
Create a vitest regression test suite that runs the full audit pipeline (25 scenarios × 4 families) on every `npm test`, catching any future builder change that breaks structural correctness, intent fidelity, or musical intelligence.

## Purpose

- Any future builder change that introduces a structural deviation, intent failure, or musical failure triggers a test failure
- All 25 scenarios covered with zero-tolerance for critical issues

<task type="auto">
  <name>Task 1: Create audit regression test suite</name>
  <files>src/lib/helix/audit-regression.test.ts</files>
  <action>
    Create a new test file that wraps the full audit pipeline as regression tests.

    **Structure:**
    ```
    describe("audit regression suite", () => {
      // Run audit once, share results across tests
      let auditResult: AuditResult;

      beforeAll(() => {
        auditResult = runAudit({ scenarios: MOCK_SCENARIOS });
      });

      it("runs all 25 scenarios without errors", () => {
        expect(auditResult.totalScenarios).toBe(25);
        // Check no scenario has an error
      });

      describe.each(["helix", "stomp", "podgo", "stadium"])("family: %s", (family) => {
        it("passes intent fidelity", () => {
          const fam = auditResult.families.find(f => f.family === family);
          expect(fam.intentSummary.failed).toBe(0);
        });

        it("passes musical intelligence", () => {
          const fam = auditResult.families.find(f => f.family === family);
          expect(fam.musicalSummary.failed).toBe(0);
        });

        it("has no pipeline errors", () => {
          const fam = auditResult.families.find(f => f.family === family);
          for (const scenario of fam.scenarios) {
            expect(scenario.error).toBeUndefined();
          }
        });
      });
    });
    ```

    **Key design choices:**
    - Use `beforeAll` to run audit once (not per-test) — ~1-2s total
    - Use `describe.each` for per-family tests — clear failure reporting
    - Test intent + musical + no-errors separately — pinpoint what broke
    - Do NOT assert on structural diff counts yet (reference corpus optional, diff counts may vary)
    - Do NOT duplicate existing mock-harness.test.ts assertions (that tests individual scenarios, this tests the aggregate audit)

    **Import from:**
    - `audit-runner.ts`: runAudit, AuditResult, AuditConfig
    - `mock-scenarios.ts`: MOCK_SCENARIOS

    Avoid: Testing individual scenario details (mock-harness.test.ts already does this). This test is for aggregate pipeline regression.
