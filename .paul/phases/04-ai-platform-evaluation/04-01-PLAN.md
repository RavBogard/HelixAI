<objective>
## Goal
Evaluate AI platform alternatives for the planner and chat roles, benchmark them against the current setup (Claude Sonnet 4.6 planner + Gemini 2.5 Flash chat), and decide whether to switch, consolidate, or keep the current dual-provider architecture.

## Purpose
The AI planner drives all creative preset decisions (amp, cab, effects, snapshots). The chat drives the user interview. Quality here directly determines whether presets "sound right" to guitarists. Phase 4 ensures we're using the best AI platform for each role — nothing off the table.

## Output
- `scripts/ai-eval-harness.ts` — Repeatable benchmark script
- `scripts/ai-eval-scenarios.ts` — Standard test conversations
- `.paul/phases/04-ai-platform-evaluation/04-01-EVAL-REPORT.md` — Evaluation report with recommendation
</objective>

## Architecture Notes
- **Two-stage architecture:** AI Planner (creative choices) → Knowledge Layer (deterministic parameters)
- **Planner output:** ToneIntent JSON — amp name, cab name, effects[], snapshots[], preset name/description
- **Knowledge Layer:** Zero AI tokens — lookup tables for all numeric parameters
- **Current planner:** Claude Sonnet 4.6 with structured output (zodOutputFormat) + 1h prompt caching
- **Current chat:** Gemini 2.5 Flash (streaming), premium tier: Gemini 3.1 Pro Preview
- **Current vision:** Claude Sonnet 4.6 (pedal photo analysis)
- **Key constraint:** Planner MUST output valid JSON matching ToneIntent Zod schema
