<objective>
## Goal
Replace the shared/generic genre-effect intelligence with per-family tailored versions that respect each device's effect budget, DSP constraints, and sonic character.

## Purpose
Currently all 4 device families receive identical effect model selection guidance (same delay, reverb, wah tables). This means a Helix user (5-7 effects, dual-DSP) gets the same recommendations as a Pod Go user (hard 4-effect limit) or a Stadium user (Agoura-era effects, arena context). Per-family effect intelligence lets the AI make smarter choices within each device's constraints.

## Output
- `genreEffectModelSection()` accepts a `DeviceFamily` parameter
- Each family gets tailored effect tables reflecting its constraints:
  - **Helix**: Full tables with dual-DSP routing context, encourages layering (e.g., dual delays, parallel reverbs)
  - **Stomp**: Tighter recommendations, prioritized by "which 3-4 effects matter most", trade-off guidance
  - **Pod Go**: Exactly 4-slot templates per genre, "what to drop first" guidance when constrained
  - **Stadium**: Agoura-era effect names where applicable, arena/FOH context in recommendations
</objective>

    - Add: "If your tone needs both delay AND reverb, choose models with shorter tails to leave DSP headroom."

    **Pod Go variant:**
    - Same genres, same model names, but restructure as "4-Slot Templates" per genre
    - Format: "Blues: Drive (Minotaur) + Delay (Transistor Tape) + Reverb (Plate) + Comp (LA Studio Comp)"
    - Each genre gets exactly one 4-effect template plus one swap suggestion
    - Add: "All 4 slots should be filled — an unused slot is wasted potential."

    **Stadium variant:**
    - Same genres but add FOH context to recommendations
    - Replace "Avoid" with "Arena Caution" (e.g., "Heavy modulation can cause phase issues through PA")
    - Add: "For arena clarity, favor Room and Hall reverbs over Spring — spring can sound harsh at volume"
    - Reference Agoura-specific effects where model names differ from HD2 catalog
    - Add note: "Stadium's headroom means effects can be run at higher mix levels without muddiness"

    **Keep backward compatible:** Default parameter `"helix"` so any missed callers still work.

    **Update tests:** Update effect-model-intelligence.test.ts to test all 4 variants produce different output and contain family-specific content.

    Avoid: Changing any model name recommendations that are correct (delays, reverbs, wahs are real Line 6 model names)
