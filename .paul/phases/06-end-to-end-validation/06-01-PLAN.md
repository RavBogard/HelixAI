## Architecture Notes
- Each DeviceTarget resolves to a DeviceFamily: helix, stomp, podgo, stadium
- Each family uses different catalogs (HELIX_AMP_NAMES vs PODGO_AMP_NAMES vs STADIUM_AMP_NAMES)
- Each family has a different builder (buildHlxFile vs buildPgpFile vs buildHspFile)
- Pod Go has 4 user effect slots, 4 snapshots, different I/O models
- Stadium uses Agoura-era amps (different model IDs), binary .hsp format
- Stomp uses 3 snapshots (Stomp) or 4 (Stomp XL), single DSP, 8 blocks max
- helix_native maps to helix family — should produce same .hlx as Floor except device ID
