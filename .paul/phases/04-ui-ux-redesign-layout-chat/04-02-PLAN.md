<objective>
## Goal
Visual redesign of the chat experience, welcome screen, and core UI components for improved readability, scannability, and professional polish — while preserving the established Studio Console dark theme.

## Purpose
Users need to quickly scan AI responses for key info (amp names, effect choices, snapshot plans). The current UI has functional components from Plan 01 but needs typography refinement, spacing improvements, and visual hierarchy upgrades to match a professional music production tool.

## Output
- Refined chat message bubbles with better visual hierarchy
- Improved typography scale and line-height for readability
- Polished welcome screen with tighter spacing
- Better suggestion chips with hover micro-interactions
- Refined input bar with integrated feel
- Improved responsive behavior at mobile breakpoints
</objective>

<acceptance_criteria>

## AC-1: Chat Typography Hierarchy
```gherkin
Given an AI response containing headings, bold text, lists, and body text
When displayed in the chat area
Then headings are visually distinct (size + weight + color differentiation)
And bold text stands out clearly against body text
And body text has minimum 16px effective size on mobile
And line-height is 1.5-1.75 for body, tighter for headings
```

## AC-2: Chat Message Visual Separation
```gherkin
Given a conversation with alternating user and assistant messages
When scrolling through the chat
Then each message has clear visual boundaries (spacing, background differentiation)
And assistant messages have a subtle background treatment (not just a left border)
And user messages are visually distinct from assistant messages
And the active/streaming message has a visible indicator
```

## AC-3: Welcome Screen Polish
```gherkin
Given a new user landing on the welcome screen
When the page loads
Then the logo, title, and subtitle have balanced vertical spacing
And the device picker has clear selected/unselected states
And suggestion chips are readable and have smooth hover transitions
And the overall layout is vertically centered with no awkward gaps
```

## AC-4: Input Bar Integration
```gherkin
Given the chat input bar at the bottom of the page
When in chat mode
Then the input bar has a subtle top border or shadow separating it from messages
And the input area has comfortable padding
And camera/send buttons maintain 44px touch targets
And the input bar is visually anchored (feels like part of the interface, not floating)
```

## AC-5: Responsive at Mobile Breakpoints
```gherkin
Given the app viewed at 375px viewport width
When viewing the welcome screen, chat, and preset card
Then no horizontal scrolling occurs
And suggestion chips stack to 2 columns (not 3)
And text remains readable (minimum 16px body)
And device picker buttons remain tappable (44px minimum)
```
