<objective>
## Goal
Fix the homescreen layout issues (logo cut off, empty right column, footer overlapping content) and visually polish the welcome screen to look professional and usable.

## Purpose
The homescreen is the first impression — right now the logo is partially cropped, there's wasted space from an empty right column, and the suggestion chips get hidden behind the footer. These issues make the app feel unfinished. Fixing them directly supports the core value of making the product accessible and trustworthy.

## Output
- Fixed homescreen layout with properly visible logo
- No wasted right-column space
- Suggestion chips fully visible above footer
- Improved visual hierarchy, spacing, and overall polish
- Device picker styled with better visual weight
</objective>

<task type="auto">
  <name>Task 1: Fix layout container and logo visibility</name>
  <files>src/app/page.tsx, src/components/WelcomeScreen.tsx, src/app/layout.tsx</files>
  <action>
    **Logo cut-off fix:**
    - The WelcomeScreen logo (280x280) is getting cut off at the top because the container uses `h-full` with `justify-center` but the content overflows vertically
    - Change WelcomeScreen's outer div from `h-full` to `min-h-full` so the content can expand beyond viewport without centering-induced cropping
    - Add top padding to ensure the logo isn't pushed above the viewport (pt-8 or similar)
    - Ensure the logo's negative-inset glow div (-48px) doesn't cause overflow issues — add overflow-visible if needed

    **Empty right column fix:**
    - The right column issue comes from the `max-w-5xl mx-auto` container in page.tsx (line 759) combined with the sidebar taking space on the left — creating asymmetric whitespace
    - The main content should center within its available space regardless of sidebar presence
    - The `max-w-5xl` (1024px) may be too wide for the welcome screen content — consider reducing to `max-w-3xl` (768px) or `max-w-2xl` (672px) for the welcome screen specifically, keeping wider for chat mode
    - Use conditional max-width: welcome screen gets a narrower max-width, chat mode keeps current width

    **Vertical layout:**
    - The welcome screen content should be vertically centered but with scroll if content exceeds viewport
    - Use `min-h-[calc(100vh-FOOTER_HEIGHT)]` or similar to account for footer
    - Add `pb-16` (or appropriate) bottom padding to ensure content clears the fixed footer

    Avoid: Changing any chat-mode layout behavior — only welcome screen layout changes
    Avoid: Removing the max-w-5xl for chat mode — only narrow the welcome screen
  </action>
