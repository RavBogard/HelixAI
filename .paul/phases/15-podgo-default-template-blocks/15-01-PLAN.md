<objective>
## Goal
Add Volume Pedal (block0), Wah (block1), and FX Loop (block4) as always-present default template blocks in Pod Go presets, matching Pod Go Edit's default layout. Add output gain snapshot controller (@controller:11).

## Purpose
Real Pod Go presets always include these three blocks at fixed positions even when disabled. Our generated presets omit them, creating structural deviations from reference presets that the audit pipeline detects.

## Output
- Updated podgo-builder.ts with template block injection
- Updated types.ts with Pod Go template block model constants
- New/updated tests verifying template blocks and output gain controller
</objective>

## Reference Preset Analysis
Three reference presets (ROCK CRUNCH, A7X, AI CHICK_ROCK) all confirm:
- block0: HD2_VolPanVolStereo (Volume Pedal) — always enabled
- block1: HD2_WahFasselStereo (Wah) — always present, usually disabled
- block4: HD2_FXLoopMono1 (FX Loop) — always present, usually disabled
- Positions vary by preset (0-1-4 vs 2-3-5), but these 3 blocks always exist
- Output block gain uses @controller:11 in controller section (some presets)
- Template blocks occupy 3 of the 10 block slots, leaving 7 for user content
</context>
