<task type="auto">
  <name>Task 1: Migrate rig-vision.ts from Claude to Gemini</name>
  <files>src/lib/rig-vision.ts</files>
  <action>
    Rewrite rig-vision.ts to use Gemini SDK for image analysis.

    **Imports:**
    - Remove: `import Anthropic from "@anthropic-ai/sdk"`
    - Add: `import { createGeminiClient, getModelId } from "@/lib/gemini"`
    - Keep: all helix imports (RigIntentSchema, types), usage-logger imports

    **API call conversion:**
    - Replace `new Anthropic({ apiKey })` with `createGeminiClient()`
    - Replace `client.messages.create()` with `ai.models.generateContent()`
    - System prompt: `config.systemInstruction` (string)
    - Image content: Gemini uses `inlineData` format in `contents` array:
      ```
      contents: [
        {
          role: "user",
          parts: [
            ...images.map(img => ({
              inlineData: { mimeType: img.mediaType, data: img.data }
            })),
            { text: "Identify all guitar/bass pedals..." }
          ]
        }
      ]
      ```
    - Max tokens: `config.maxOutputTokens: 1024`
    - Model: Use `getModelId(false)` (standard tier — gemini-2.5-flash)

    **System prompt:**
    - Keep VISION_SYSTEM_PROMPT content identical (same pedal extraction instructions)
    - Move from Anthropic `system:` parameter to Gemini `config.systemInstruction`

    **Response parsing:**
    - Replace `response.content.find(block => block.type === "text").text` with `response.text`
    - Keep all three JSON extraction strategies (direct parse, code fence, substring)
    - Keep RigIntentSchema.parse() Zod validation

    **Usage logging:**
    - Replace `estimateClaudeCost` with `estimateGeminiCost`
    - Update token fields: promptTokenCount, candidatesTokenCount, etc.
    - Update console.log format

    **Env var:**
    - Remove `CLAUDE_API_KEY` check (createGeminiClient handles GEMINI_API_KEY internally)

    **Function name:**
    - Keep `callRigVisionPlanner` name (it's the vision planner, not the tone planner)
    - Update JSDoc to mention Gemini instead of Claude

    Avoid: Changing the RigIntent Zod schema.
    Avoid: Changing the vision route (route.ts).
    Avoid: Changing the VISION_SYSTEM_PROMPT content (same extraction instructions).
  </action>
  <verify>npx tsc --noEmit passes; grep "@anthropic-ai/sdk" src/lib/rig-vision.ts returns no matches</verify>
  <done>AC-1 and AC-4 satisfied: Vision uses Gemini SDK, TypeScript compiles clean</done>
</task>
