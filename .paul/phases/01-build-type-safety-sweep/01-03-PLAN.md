<task type="auto">
  <name>Task 1: Migrate planner.ts from Claude to Gemini</name>
  <files>src/lib/planner.ts</files>
  <action>
    Rewrite planner.ts to use Gemini SDK. Use the eval harness pattern (scripts/ai-eval-harness.ts:420-470) as reference.

    **Imports:**
    - Remove: `import Anthropic from "@anthropic-ai/sdk"` and `import { zodOutputFormat } from "@anthropic-ai/sdk/helpers/zod"`
    - Add: `import { createGeminiClient } from "@/lib/gemini"` (existing helper)
    - Keep: all helix imports, usage-logger imports, prompt-router import

    **Function signature:**
    - Rename `callClaudePlanner` → `callGeminiPlanner`
    - Keep same parameters: `(messages, device?, family?, toneContext?)`
    - Keep same return type: `Promise<ToneIntent>`

    **API call conversion:**
    - Replace `new Anthropic({ apiKey })` with `createGeminiClient()`
    - Replace `client.messages.create()` with `ai.models.generateContent()`
    - System prompt: `config.systemInstruction` (string, not array)
    - User content: `contents` (string)
    - Structured output: `config.responseMimeType: "application/json"` + `config.responseJsonSchema`
    - Max tokens: `config.maxOutputTokens: 4096`
    - Remove `cache_control` / ephemeral cache (Gemini handles caching differently)

    **JSON Schema for structured output:**
    - Gemini requires a plain JSON Schema object, not Zod.
    - Use `zodToJsonSchema` from the `zod-to-json-schema` npm package to convert the Zod schema.
    - If `zod-to-json-schema` is not installed, install it as a devDependency.
    - Alternative: use a manual JSON schema builder like `buildGeminiJsonSchema` in the eval harness (scripts/ai-eval-harness.ts:256-297). However, that approach duplicates the schema. Prefer `zod-to-json-schema` for DRYness since the Zod schema is the source of truth.
    - IMPORTANT: Gemini's structured output may not support all JSON Schema features (e.g., `$defs`, `$ref`). If `zod-to-json-schema` produces incompatible output, fall back to the manual approach from the eval harness.

    **Response parsing:**
    - Replace `response.content.find(block => block.type === "text").text` with `response.text`
    - Keep JSON.parse + sanitization (snapshot name truncation, variaxModel stripping)
    - Keep final `getToneIntentSchema(family).parse(raw)` Zod validation

    **Usage logging:**
    - Replace `estimateClaudeCost(usage)` with `estimateGeminiCost(response.usageMetadata, modelId)`
    - Update the logUsage record:
      - `model`: use Gemini model ID (from `getModelId(false)` — standard tier for planner)
      - `input_tokens`: `usageMetadata.promptTokenCount`
      - `output_tokens`: `usageMetadata.candidatesTokenCount`
      - `cache_creation_input_tokens`: null (Gemini doesn't report this)
      - `cache_read_input_tokens`: `usageMetadata.cachedContentTokenCount`
      - `total_tokens`: `usageMetadata.totalTokenCount`
    - Update console.log format to match Gemini fields

    **Stomp restriction:**
    - Keep the stompRestriction logic — append to user content, same as before

    **Env var:**
    - Remove `CLAUDE_API_KEY` check (createGeminiClient handles GEMINI_API_KEY internally)
    - Note: CLAUDE_API_KEY is still needed for rig-vision.ts — do NOT remove from .env

    Avoid: Changing rig-vision.ts (stays on Claude for image processing).
    Avoid: Changing the ToneIntent Zod schema.
    Avoid: Removing the Anthropic SDK from package.json (rig-vision still needs it).
  </action>
