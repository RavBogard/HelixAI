    - All `(s) => s.fieldName` selectors: type `s` as `VisualizerStoreState` from `./store`
    - `store.ts` line 101: `(set, get)` — type as zustand's `StoreApi` set/get, or use the typed `create<VisualizerStoreState>()` generic which infers them
    - `store.ts` line 118: `hydrate(device, baseBlocks, ...)` — add types matching VisualizerStoreState fields
    - Other action methods: `selectSnapshot(index: number)`, `selectBlock(blockId: string | null)`, etc.
    - `SignalChainCanvas.tsx` lines 374, 395, 398: `(b)` parameters — type as `BlockSpec`
    - `SnapshotSelectorBar.tsx` line 19: `(snap, index)` — type as `SnapshotSpec, number`

    **Test files (store.test.ts, *.test.tsx):**
    - Store selectors: same `VisualizerStoreState` typing
    - Array callbacks like `.find((b) =>)`, `.filter((b) =>)`, `.sort((a, b) =>)`: type as `BlockSpec`
    - `.map((b, i) =>)`: type as `BlockSpec, number`

    Import `VisualizerStoreState` from the store module where needed.
    Import `BlockSpec`, `SnapshotSpec` from `@/lib/visualizer/types` where needed.

    Avoid: Using `any` type — always use the specific interface.
    Avoid: Changing any runtime logic — this is types-only.
