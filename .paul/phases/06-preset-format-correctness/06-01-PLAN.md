<objective>
## Goal
Audit and fix all preset file format correctness issues across every device builder —
DSP routing, footswitch/stomp assignments, block type IDs, and I/O model values —
by reverse-engineering correct values from the real .hlx/.pgp/.hsp format specification
and validating against HX Edit.

## Purpose
The app generates presets with structural bugs that make them unusable or incorrect
when loaded into HX Edit. These are not cosmetic — they produce broken signal chains
(no audio path from DSP0→DSP1) and missing stomp control (effects can't be toggled live).
Fixing these comprehensively ensures every generated preset is structurally valid across
all 10 device targets.

## Output
- All four builders (preset-builder, stomp-builder, podgo-builder, stadium-builder) produce
  structurally correct preset files
- DSP routing connects DSP0→DSP1 on dual-DSP devices
- Every toggleable effect block gets a footswitch assignment (device-aware)
- Block @type values validated against real firmware
</objective>

## Known Issues Found in Audit
1. **DSP routing broken (preset-builder.ts:143):** `dsp0.outputA.@output: 1` always routes
   to physical 1/4" output. When DSP1 has blocks, this should route to DSP1 (Flow 2 Input).
   Result: signal never reaches DSP1 blocks (delays, reverbs, EQ, gain block).

2. **Stomp assignments limited to 4 (preset-builder.ts:546):** `STOMP_FS_INDICES = [7, 8, 9, 10]`
   hardcodes only 4 footswitch slots. The Helix LT has FS1-FS8; the Floor has FS1-FS12.
   Most effects go unassigned and can't be toggled during performance.

3. **Stomp builder has ZERO footswitch assignments (stomp-builder.ts:392):**
   `footswitch: { dsp0: {}, dsp1: {} }` — no blocks get stomp assignments on HX Stomp/Stomp XL.

4. **Block @type for modulation = 4 (preset-builder.ts:645):** Same value as cab type.
   Needs verification against real .hlx files — may cause HX Edit to misidentify blocks.

5. **No device-aware footswitch indices:** All devices use the same STOMP_FS_INDICES.
   Different devices have different footswitch counts and index ranges.
</context>
