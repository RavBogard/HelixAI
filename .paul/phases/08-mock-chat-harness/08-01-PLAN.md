<objective>
## Goal
Create a mock preset generation harness that exercises the full Knowledge Layer pipeline (assembleSignalChain → resolveParameters → buildSnapshots → validate → build*File) across all 5 device families and multiple tone scenarios, producing real preset output files for automated comparison in later phases.

## Purpose
The v5.0 automated gold standard compliance pipeline needs a way to generate presets deterministically (no AI calls, no cost) for structured comparison against reference presets. This harness is the generation infrastructure that all subsequent audit phases depend on.

## Output
- `src/lib/helix/mock-scenarios.ts` — ToneIntent fixtures for 5 device families × 5 tone styles (25 scenarios)
- `src/lib/helix/mock-harness.ts` — Harness function that runs scenarios through full pipeline, returns structured results
- `src/lib/helix/mock-harness.test.ts` — Tests proving all 25 scenarios produce valid preset files
</objective>

  <name>Task 1: Create mock scenario fixtures</name>
  <files>src/lib/helix/mock-scenarios.ts</files>
  <action>
    Create a module exporting mock ToneIntent fixtures organized by device family and tone style.

    Structure:
    ```
    interface MockScenario {
      id: string;                    // e.g., "helix-clean", "podgo-bass"
      device: DeviceTarget;          // e.g., "helix_floor", "pod_go"
      toneStyle: string;             // "clean" | "high-gain" | "blues" | "ambient" | "bass"
      intent: ToneIntent;            // Full ToneIntent with valid catalog names
    }
    export const MOCK_SCENARIOS: MockScenario[];
    ```

    For each of the 5 families, create 5 scenarios:
    - **Clean:** Single-coil, clean amp (e.g., US Deluxe Nrm), chorus/reverb, 4 snapshots
    - **High-gain:** Humbucker, high-gain amp (e.g., Placater Dirty), distortion/delay, 4 snapshots
    - **Blues/Crunch:** Single-coil or humbucker, crunch amp (e.g., Brit Plexi Brt), overdrive/reverb
    - **Ambient:** Clean amp, heavy delay/reverb, tempo hint, ambient snapshot
    - **Bass:** Instrument "bass", bass amp (from bass catalog), compression always included

    Use ONLY amp/cab/effect names from the correct family catalog (HELIX_AMP_NAMES, STOMP_AMP_NAMES, PODGO_AMP_NAMES, STADIUM_AMP_NAMES, etc.). Import from @/lib/helix.

    For Stomp/StompXL: limit effects to maxEffects from STOMP_CONFIG.
    For PodGo: limit effects appropriately (4-slot templates).
    For Stadium: use Agoura-era amp names.

    Each scenario snapshot array should have 4 entries:
    - { name: "Clean", toneRole: "clean" }
    - { name: "Crunch", toneRole: "crunch" }
    - { name: "Lead", toneRole: "lead" }
    - { name: "Ambient", toneRole: "ambient" }

    Avoid: Using amp/effect names not in the catalog (will fail validation). Using generic names — use exact catalog strings.
  </action>
  <verify>
    Import MOCK_SCENARIOS and verify length >= 25.
    For each scenario, verify intent.ampName is in the correct family's amp catalog.
  </verify>
  <done>AC-1 satisfied: 25+ scenarios across 5 families × 5 styles, AC-4 satisfied: bass scenarios have instrument="bass" and compression</done>

<task type="auto">
  <name>Task 2: Create mock harness runner and tests</name>
  <files>src/lib/helix/mock-harness.ts, src/lib/helix/mock-harness.test.ts</files>
  <action>
    **mock-harness.ts:**
    Create a function that takes a MockScenario and runs the full pipeline:

    ```
    interface HarnessResult {
      scenarioId: string;
      device: DeviceTarget;
      toneStyle: string;
      preset: unknown;              // HlxFile | PgpFile | HspFile
      spec: PresetSpec;
      fileExtension: ".hlx" | ".pgp" | ".hsp";
      qualityWarnings: string[];
      intentAudit: IntentAudit;
      error?: string;
    }

    function runScenario(scenario: MockScenario): HarnessResult;
    function runAllScenarios(): HarnessResult[];
    ```

    Pipeline (mirrors generate/route.ts without auth/Supabase):
    1. getCapabilities(scenario.device)
    2. assembleSignalChain(scenario.intent, caps)
    3. resolveParameters(chain, scenario.intent, caps)
    4. buildSnapshots(parameterized, scenario.intent.snapshots)
    5. Build PresetSpec with name from scenario id
    6. validatePresetSpec(spec, caps)
    7. validatePresetQuality(spec, caps)
    8. auditIntentFidelity(scenario.intent, spec)
    9. Build file: isStomp → buildStompFile, isStadium → buildHspFile, isPodGo → buildPgpFile, else → buildHlxFile
    10. Return HarnessResult

    Wrap each scenario in try/catch — set error field on failure, don't throw.

    **mock-harness.test.ts:**
    Test that all scenarios pass:
    - `describe("mock-harness")` with `it.each` over MOCK_SCENARIOS
    - Each scenario: runScenario() succeeds (no error)
    - Each scenario: preset has correct device ID (from DEVICE_IDS)
    - Each scenario: fileExtension matches family
    - Aggregate test: runAllScenarios() returns 25+ results with zero errors
    - Bass scenarios: verify instrument field propagated

    Avoid: Making AI calls. Importing anything from @/lib/gemini or @/lib/planner.
  </action>
