<objective>
## Goal
Reduce token consumption and cost per preset generation without quality regression. Add chat history truncation, tune maxOutputTokens, and create a cost analysis script for ongoing monitoring.

## Purpose
The system already has smart caching and usage logging, but lacks: (1) chat history length limits — unbounded conversation history is sent to the planner, inflating input tokens on long sessions; (2) maxOutputTokens is set to 4096 but ToneIntent JSON output is typically ~300-500 tokens — the excess reserves capacity that Gemini charges for in thinking budget; (3) no tooling to analyze accumulated usage.jsonl data for cost hotspots.

## Output
- Chat history truncated to last N messages before planner call (reduces input tokens)
- maxOutputTokens reduced from 4096 to 2048 (ToneIntent JSON never exceeds ~600 tokens)
- Cost analysis script that reads usage.jsonl and reports per-endpoint/per-device/cache-hit breakdowns
- All 1455 existing tests pass with zero regressions
</objective>
