<acceptance_criteria>

## AC-1: Chat Message Accessibility
```gherkin
Given a screen reader user viewing the chat interface
When the AI streams a new message
Then the message container has aria-live="polite" so the screen reader announces new content
And each message has role="article" with aria-label identifying sender
```

## AC-2: Interactive Control Labels
```gherkin
Given a screen reader user navigating the interface
When they tab to any interactive button (camera, send, analyze, suggestion chips, device picker, download)
Then the button has an aria-label describing its action
And toggle-style controls (device picker) have aria-pressed reflecting current state
```

## AC-3: Semantic Landmarks
```gherkin
Given a keyboard-only user
When they navigate via landmark shortcuts
Then the footer has a nav or contentinfo role
And ModelBrowserDropdown categories have aria-expanded
And the main chat area is identifiable as main content
```

## AC-4: Color Contrast Compliance
```gherkin
Given the dark theme with --hlx-text-muted (#524840) on --hlx-surface (#161513)
When tested with WCAG 2.1 AA contrast ratio (4.5:1 for normal text)
Then all text meets the minimum ratio, or muted text colors are adjusted upward
```

## AC-5: Test Infrastructure Normalized
```gherkin
Given a developer running tests
When they execute `npm test`
Then vitest runs with jsdom environment by default
And per-file @vitest-environment overrides are removed
And the React Compiler warning in ChatInput is resolved
```
